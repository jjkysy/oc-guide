# 测试用例：009 - 故障排查

> 对应指南 [009 - 故障排查与注意事项](../guide/009-troubleshooting.md)

## TC-009-01：openclaw doctor 诊断

**操作步骤**：
1. 执行 `openclaw doctor`

**测试目标**：验证诊断工具覆盖常见问题

**预期结果**：
- 输出各项检查结果（Node.js 版本、配置文件、API 连接等）
- 给出具体的修复建议
- 可自动修复的问题标注为已修复

---

## TC-009-02：配置严格验证错误

**操作步骤**：
1. 手动在 `openclaw.json` 中添加 `"fakeKey": "value"`
2. 执行 `openclaw start`

**测试目标**：验证严格验证的错误提示

**预期结果**：
- Gateway 拒绝启动
- 错误信息明确指出未知键 `fakeKey`

---

## TC-009-03：API Key 失效处理

**操作步骤**：
1. 设置一个无效的 API Key
2. 启动 Gateway
3. 发送测试消息

**测试目标**：验证 API 认证失败的错误处理

**预期结果**：
- 日志中记录 401 Unauthorized 错误
- 代理不会崩溃，返回错误提示
- 配置正确的 Key 后恢复正常

---

## TC-009-04：日志排查

**操作步骤**：
1. 制造一个错误（如无效的 Bot Token）
2. 执行 `openclaw logs --tail 50`
3. 检查日志中的错误信息

**测试目标**：验证日志输出对排错的帮助

**预期结果**：
- 日志中包含时间戳、错误级别和详细信息
- 能定位到具体的错误原因
